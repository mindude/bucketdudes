4. Explain how routes work in angular and a code example.10 points.

What Angular does with routes is that instead of loading each page individually, it loads the whole thing, so when you do an action in the webpage, the application doesn't have to reload again.
In code, this is how a route is established in the page:

<a ng-href="#/route is matched">Homework 1</a>

When a route is called, angular first checks the routes that are in the app.js in the .config(function ($routeProvider) { $routeProvider)}, then it tries to match the submitted route with the ones available (.when('/route is matched', {}), in case nothing is matched, if written in code, the path redirects to the one written in .otherwise({redirectTo: '/redirection'}), if the otherwise is not written in code, Angular just tries to load the view (but loads nothing).
Finally, inside of the {} in the .when, it is displayed a templateUrl, which searches where the view is inside the folders of the project, controller who searches inside the controllers who has that name assigned, and if assigned the ControllerAs line, then the controller instance will be assigned to a property on the new scope.

Example of the code app.js:

.config(function ($routeProvider) {
$routeProvider .when('/', { templateUrl: 'views/main.html', controller: 'MainCtrl', controllerAs: 'main' })
.when('/about', { templateUrl: 'views/about.html', controller: 'AboutCtrl', controllerAs: 'about' })
.when('/tareas', { templateUrl: 'views/tareas.html', controller: 'TareasCtrl', controllerAs: 'tareas' })
.when('/homework1', { templateUrl: 'views/homework1.html', controller: 'Homework1Ctrl', controllerAs: 'homework1' })
.when('/homeworkindex', { templateUrl: 'views/homeworkindex.html', controller: 'HomeworkindexCtrl', controllerAs: 'homeworkindex' })
.otherwise({ redirectTo: '/' }); });
